Vol.10 — グラフィックデザイン / 1961

デザインに
「正解の格子」を
引いた男

1961 Grid Systems in Graphic Design / Josef Müller-Brockmann

「センス」でデザインしていた時代に、「グリッド」という数学的なルールを持ち込んだ男がいた。その発明はWebデザインからスマホアプリまで、現代のすべての画面に生きている。

Work
Grid Systems in Graphic Design, 1961 — Josef Müller-Brockmann Niggli Verlag — 引用

1961年以前、グラフィックデザインは職人の「勘」と「センス」に依存していた。どこに文字を置くか、余白をどれだけとるか——すべてデザイナーの経験と感性で決まっていた。再現性がなく、複数人で作業すると必ずバラバラになった。

スイスのデザイナーJosef Müller-Brockmannはそこに問いを立てた。「デザインの配置を、数学的に決められないか」。彼が提唱したのはグリッドシステム——ページを均等な格子状に分割し、すべての要素をその格子に沿って配置するというルールだった。

衝撃的だったのはその思想だ。グリッドは制約ではなく自由だ、とMüller-Brockmannは言った。ルールがあるから、誰でも同じ品質を出せる。ルールがあるから、大量の情報を整理できる。ルールがあるから、異なるページが一冊の本として統一感を持つ。この発想はのちにWebデザインの基礎となり、BootstrapやTailwindのカラムシステム、iOSのレイアウトガイドラインに直接受け継がれている。

グリッドシステムの3原則
01
秩序（Order）
すべての要素を格子に沿って配置する。「なんとなく」の配置を排除し、数学的根拠のある位置決めをする。
02
一貫性（Consistency）
同じグリッドを使えば、複数のデザイナーが作業しても統一感が生まれる。チームでのデザインを可能にした。
03
明快さ（Clarity）
整理された配置は情報の優先順位を明確にする。読者が「何を先に読むか」を迷わない構造を作る。

この3原則はそのままWebのCSSグリッド、iOSのHuman Interface Guidelines、Material Designに受け継がれている。

"The grid system is an aid, not a guarantee. It permits a number of possible uses and each designer can look for a solution appropriate to his personal style."

「グリッドシステムは手助けであって、保証ではない。多くの可能性を許容し、各デザイナーは自分のスタイルに合った解決策を見つけられる。」

— Josef Müller-Brockmann
